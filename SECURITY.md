# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| Latest (main) | ✅ |

## Reporting a Vulnerability

If you discover a security vulnerability in Swahit, please **do not** open a public GitHub issue.

Instead, email us privately at: **security@swahit.app**

Include:
- A clear description of the vulnerability
- Steps to reproduce it
- Potential impact
- Any suggested fixes (optional)

We will respond within 48 hours and work with you to resolve the issue responsibly.

## What We Consider a Vulnerability

- Unauthorized access to user data or conversations
- Authentication/authorization bypass
- SQL injection or data leakage
- Exposed API keys or secrets in code
- XSS or CSRF vulnerabilities in the frontend

## Responsible Disclosure

We ask that you:
- Give us reasonable time to patch before public disclosure
- Not access or modify data belonging to other users
- Act in good faith

Thank you for helping keep Swahit and our users safe.
